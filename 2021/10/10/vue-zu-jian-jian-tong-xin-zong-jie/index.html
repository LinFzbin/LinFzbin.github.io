vue组件间通信总结

组件间通信总结

1.props配置项 （父子组件间通信）

  1. 功能：让组件接收外部传过来的数据

  2. 传递数据：

    <Demo :name="xxx"/>

    ​ 这样直接传的就是String类型的数据

  3. 接收数据：

    1. 第一种方式（只接收）：

      props:['name']

  1. 第二种方式（限制类型）：

    props:{name:String}

  1. 第三种方式（限制类型、限制必要性、指定默认值）：

    // props对象
props:{
	name:{
	type:String, // 传递数据类型
	required:true, // 设置必须传递数据 
	default:'老王' // 默认值
	}
}

    备注：props是只读的，Vue底层会监测你对props的修改，如果进行了修改，就会发出警告，若业务需求确实需要修改，那么请复制props的内容到data中一份，然后去修改data中的数据。

2.自定义事件（父子组件间通信）

  1. 绑定自定义事件：

    1. 第一种方式，在父组件中：

      // 简写
<Demo @zdy="test"/>  
// v-on写法
<Demo v-on:zdy="test"/>

    2. 第二种方式，在父组件中：

      相当于借助了ref属性

      // 给Demo组件通过 ref 打上一个标签
<Demo ref="demo"/>
......
mounted(){
   // 这里就是给 Demo 组件绑定一个自定义事件
   this.$refs.demo.$on('zdy',this.test)
}

    3. 若想让自定义事件只能触发一次，可以使用once修饰符，或$once方法。

      <Demo @zdy.once="test"/>

  1. 子组件中触发自定义事件：

    this.$emit('zdy',数据)

    ​

  2. 子组件中解绑自定义事件

    this.$off('zdy')

  1. 组件上也可以绑定原生DOM事件，需要使用native修饰符。

    <Demo @click.native="test"/>

  2. 注意：通过绑定自定义事件时，回调要么配置在methods中，要么用箭头函数，否则this指向会出问题！

    this.$refs.demo.$on('zdy',回调)

原生dom事件和自定义事件

组件标签上添加的所有事件都是自定义事件，html标签上添加的事件都是原生的dom事件。

  1. 组件上也可以绑定原生DOM事件，需要使用native修饰符。

    <Demo @click.native="test"/>

3.v-model （父子组件间通信）

本质：其实就是单向数据绑定 v-bind（读）和 input事件（写）的结合，适用于表单类元素（input、select、textarea等）

<!-- v-model 的本质是 单向数据绑定 v-bind 和 input事件(原生的dom事件)的结合 -->
<input type="text" :value="name" @input="name = $event.target.value"><br/>

用在组件上时，这里的子组件是CustomInput

// 父组件中 :value="name" 是 props 传参  
// @input 这里指的是自定义事件 input
<CustomInput :value="name" @input="name = $event"></CustomInput>
// 直接使用 v-model 这里props传递的属性名默认为 value 
<CustomInput v-model="name"></CustomInput> <hr/>
    
    
// 子组件中   $emit触发自定义事件 并传递当前 input 输入的值
<input type="text" :value="value" @input="$emit('input',$event.target.value)"> 
...
// 这里的value需要用props接收属性名为 value
props: ['value']

4.sync（父子组件间通信）

// 父组件中
// 不使用sync修改符
// 传递的参数是 money 绑定的自定义事件为 @update:money 这里自定义事件必须这么写
// 这里的 $event 其实就是传过来的参数
<Child :money="money" @update:money="money = $event"></Child>

// 使用sync修改符
<Child :money.sync="money"></Child>

// 子组件中
// 在button标签上 触发自定义事件 update:money
<button @click="$emit('update:money',money-100)">花钱</button>

v-model和.sync的区别

两者都能达到父子组件间数据通信，v-model绑定的子组件内部一定是表单元素（例如：input、select、textarea等）；而sync绑定的子组件内部不是表单元素，如上面绑定的就是一个button按钮。

5.$attrs和$listeners （父子组件间通信）

$attrs：父组件传递给子组件所有属性组成的对象（除了props接收的以及style和class）

$listeners：父组件传递给子组件所有事件监听组成的对象

一般用上面二者对一个组件实现二次封装，例如下面就是对element-ui的button组件进行二次封装。

<a href="javascript:;" :title="title" style="margin-left: 15px">
	// 这里的 v-bind 和 v-on 必须写完整形式  不能简写
	<el-button v-bind="$attrs" v-on="$listeners"></el-button>
</a>

6.$ref、$parent 和 $children（父子组件间通信）

  • $children：所有子组件对象的组成的数组​
  • $parent： 代表父组件对象

上面二者都不介意使用，this.$children[0] 不知道其指代的是哪个子组件对象；this.$parent 可能当前子组件的父组件有多个。

mixin（混入）

也就是js代码相同，将其进行封装，提高代码复用；

新建一个mymixin.js文件 在js文件中暴露一个对象 对象内部可以有data methods computed… 会将js文件中暴露出的数据 方法等混入到组件内部
在组件内部引入 import mymixin from ‘./myminxi.js’
使用 mixins:[mymixin]

7.插槽（父子组件间通信）

理解：父组件填自组卷挖的坑，相当于是一个填坑的操作。

插槽分为三种：默认插槽、具名插槽和作用域插槽（这个用的最多），一般只有一个默认插槽，有多个插槽的话使用具名插槽。

默认插槽

//父组件中：
<Category>
	<div>html结构</div>
</Category>

//子组件中：
<template>
<div>
	// 定义插槽 没有使用该默认插槽则显示 当前默认插槽的 默认文字
	<slot>插槽默认内容...</slot>
</div>
</template>

具名插槽

//父组件中：
        <Category>
            // 具名插槽和template配合使用 绑定 一个具名插槽 center
            <template v-slot="center">
              <div>html结构1</div>
            </template>
			// 绑定 一个具名插槽 footer
            <template v-slot:footer>
               <div>html结构2</div>
            </template>
        </Category>
//子组件中：
        <template>
            <div>
               <!-- 定义插槽 -->
               <slot name="center">插槽默认内容...</slot>
               <slot name="footer">插槽默认内容...</slot>
            </div>
        </template>

作用域插槽

这里直接用一个栗子来理解，当然这里是为了举例，真实情况是不清楚子组件中的数据，但是需要进行数据的回传，由父组件决定其展示的结构

  1. 数据在父组件中 在子组件中进行数据的展示
  2. 父组件 传递数据 子组件 props 接收 v-for 对数据进行展示
  3. 数据结构由父组件决定，所以需要将每一个数据回传到父组件中
// 显示TODO列表时, 已完成的TODO为绿色
// 父组件
<!-- 先将数据从父组件传递到子组件 -->
<List :todos="todos">
    <!-- slot-scope就是作用域插槽   myScopeProps接收的是子组件回传过的对象  这里指的是接收子组件回传过来的一个个todo对象 -->
    <template slot-scope="myScopeProps">
    	<!-- style样式动态绑定 -->
    	<span :style="{color: myScopeProps.todo.isComplete?'green':'red'}">
        {{myScopeProps.todo.text}}
   	 	</span>
    </template>
</List>


// 子组件
<ul>
    <li v-for="(todo, index) in todos" :key="index">
      <!-- 这里不是props传递 是作用域插槽的子传父 -->
      <slot :todo="todo" :index="index">
          {{todo.text}}  // 这个展示的todo 是v-for展示由父组件传过来的数据
      </slot>
    </li>
</ul>
// 显示TODO列表时, 带序号, TODO的颜色为蓝绿搭配
<List :todos="todos">
        <!-- 这个是自己写的  相当于myScopeProps 接收的是todo的对象集合 同上 -->
        <!-- <template slot-scope="myScopeProps">
          <h2 :style="{color: myScopeProps.todo.isComplete?'blue':'green'}">
            {{myScopeProps.todo.id}} --- {{myScopeProps.todo.text}}
          </h2>
        </template> -->
        <!-- 这里相当于结构赋值 -->
        <template slot-scope="{todo,index}">
          <h2 :style="{color: todo.isComplete?'green':'blue'}">
              <span>{{index+1}}</span> --- {{todo.text}}
          </h2>
        </template>
</List>

8.vuex（所有组件间通信）

一般看项目大小决定是否使用vuex

首先是先安装vuex插件

npm  i vuex

  1. 创建文件：src/store/index.js

    // vuex的store
// 引入vue
import Vue from 'vue'
// 引入vuex插件
import Vuex from 'vuex'

// 使用插件Vuex
Vue.use(Vuex)

// actions 响应组件中的动作
const actions = {}
// 用于操作数据
const mutations = {}
// 用于存储数据
const state = {}

// 创建并暴露store
export default new Vuex.Store({
    actions,
    mutations,
    state,
    })

  2. 在main.js中创建vm时传入store配置项

    ......
//引入store
import store from './store'
......

// 创建vm
new Vue({
  el: '#app',
  render: h => h(App),
  store: store,
})

基本使用

  1. 初始化数据、配置actions、配置mutations，操作文件store.js

    //引入Vue核心库
import Vue from 'vue'
//引入Vuex
import Vuex from 'vuex'
//引用Vuex
Vue.use(Vuex)

const actions = {
    //响应组件中加的动作
	jia(context,value){
		// console.log('actions中的jia被调用了',miniStore,value)
		context.commit('JIA',value)
	},
}

const mutations = {
    //执行加
	JIA(state,value){
		// console.log('mutations中的JIA被调用了',state,value)
		state.sum += value
	}
}

//初始化数据
const state = {
   sum:0
}

// 创建 并 暴露 store
export default new Vuex.Store({
	actions,
	mutations,
	state,
})

  2. 组件中读取vuex中的数据：$store.state.sum

  3. 组件中修改vuex中的数据：$store.dispatch(‘action中的方法名’,数据) 或 $​store.commit(‘mutations中的方法名’,数据)

    备注：若没有网络请求或其他业务逻辑，组件中也可以越过actions，即不写dispatch，直接编写commit

getters的使用

  1. 概念：当state中的数据需要经过加工后再使用时，可以使用getters加工。

  2. 在store.js中追加getters配置

    ......

const getters = {
	bigSum(state){
		return state.sum * 10
	}
}

//创建并暴露store
export default new Vuex.Store({
	......
	getters
})

  3. 组件中读取数据：$store.getters.bigSum

四个map方法的使用

  1. mapState方法：用于帮助我们映射state中的数据为计算属性

    computed: {
    //借助mapState生成计算属性：sum、school、subject（对象写法）
     ...mapState({sum:'sum',school:'school',subject:'subject'}),
         
    //借助mapState生成计算属性：sum、school、subject（数组写法）
    ...mapState(['sum','school','subject']),
},

  2. mapGetters方法：用于帮助我们映射getters中的数据为计算属性

    computed: {
    //借助mapGetters生成计算属性：bigSum（对象写法）
    ...mapGetters({bigSum:'bigSum'}),

    //借助mapGetters生成计算属性：bigSum（数组写法）
    ...mapGetters(['bigSum'])
},

  3. mapActions方法：用于帮助我们生成与actions对话的方法，即：包含$store.dispatch(xxx)的函数

    methods:{
    //靠mapActions生成：incrementOdd、incrementWait（对象形式）
    ...mapActions({incrementOdd:'jiaOdd',incrementWait:'jiaWait'})

    //靠mapActions生成：incrementOdd、incrementWait（数组形式）
    ...mapActions(['jiaOdd','jiaWait'])
}

  4. mapMutations方法：用于帮助我们生成与mutations对话的方法，即：包含$store.commit(xxx)的函数

    methods:{
    //靠mapActions生成：increment、decrement（对象形式）
    ...mapMutations({increment:'JIA',decrement:'JIAN'}),
    
    //靠mapMutations生成：JIA、JIAN（对象形式）
    ...mapMutations(['JIA','JIAN']),
}

备注：mapActions与mapMutations使用时，若需要传递参数需要：在模板中绑定事件时传递好参数，否则参数是事件对象。

模块化+命名空间

  1. 目的：让代码更好维护，让多种数据分类更加明确。

  2. 修改store.js

    const countAbout = {
  namespaced:true,//开启命名空间
  state:{x:1},
  mutations: { ... },
  actions: { ... },
  getters: {
    bigSum(state){
       return state.sum * 10
    }
  }
}

const personAbout = {
  namespaced:true,//开启命名空间
  state:{ ... },
  mutations: { ... },
  actions: { ... }
}

const store = new Vuex.Store({
  // 模块化
  modules: {
    countAbout,
    personAbout
  }
})

    这里相当于可以将countAbout和personAbout单独写一个js文件，在store.js中将其引入

  3. 开启命名空间后，组件中读取state数据：

    //方式一：自己直接读取
this.$store.state.personAbout.list
//方式二：借助mapState读取：
...mapState('countAbout',['sum','school','subject']),

  4. 开启命名空间后，组件中读取getters数据：

    //方式一：自己直接读取
this.$store.getters['personAbout/firstPersonName']
//方式二：借助mapGetters读取：
...mapGetters('countAbout',['bigSum'])

  5. 开启命名空间后，组件中调用dispatch

    //方式一：自己直接dispatch
this.$store.dispatch('personAbout/addPersonWang',person)
//方式二：借助mapActions：
...mapActions('countAbout',{incrementOdd:'jiaOdd',incrementWait:'jiaWait'})

  6. 开启命名空间后，组件中调用commit

    //方式一：自己直接commit
this.$store.commit('personAbout/ADD_PERSON',person)
//方式二：借助mapMutations：
...mapMutations('countAbout',{increment:'JIA',decrement:'JIAN'}),

9.全局事件总线（所有组件间通信）

  1. 一种组件间通信的方式，适用于任意组件间通信。

  2. 安装全局事件总线：

    new Vue({
	......
	beforeCreate() {
		Vue.prototype.$bus = this // 借助生命周期钩子beforeCreate安装全局事件总线，$bus就是当前应用的vm
	},
    ......
})

  3. 使用事件总线：

    1. 接收数据：A组件想接收数据，则在A组件中给$bus.$on 绑定自定义事件，事件的回调留在A组件自身。

      mounted() {
    this.$bus.$on('checkTodo',this.checkTodo)

    this.$bus.$on('deleteTodo',this.deleteTodo)
},
    
...
methods:{
    // 回调函数留在当前组件自身
    checkTodo（id）{
   	... 
    }
}

    2. 提供数据，其实就是触发自定义事件：this.$bus.$​emit(‘xxxx’,数据)

      methods: {
    // 切换勾选状态
    handleCheck(id){
        this.$bus.$emit('checkTodo',id)
    },
    // 删除一个todo
    handleDelete(id){
        if(confirm('are you sure?')){
           this.$bus.$emit('deleteTodo',id)
        }
    }
},

  1. 最好在beforeDestroy钩子中，用$off去解绑当前组件所用到的事件。

    beforeDestroy(){
    this.$bus.$off('checkTodo')
    this.$bus.$off('deleteTodo')
}

10.pubsub（所有组件间通信）

  1. 一种组件间通信的方式，适用于任意组件间通信。

  2. 使用步骤：

    1. 安装 pubsub：npm i pubsub-js

    2. 引入: import pubsub from ‘pubsub-js’

    3. 接收数据：A组件想接收数据，则在A组件中订阅消息，订阅的回调留在A组件自身。

      methods(){
  demo(data){......}
}
......
mounted() {
  // 相当于事件总线中的 绑定自定义事件 
  this.pid = pubsub.subscribe('xxx',this.demo) //订阅消息
}

    4. 提供数据：pubsub.publish(‘xxx’,数据)，相当于事件总线中的触发自定义事件

    5. 最好在beforeDestroy钩子中，用PubSub.unsubscribe(pid)去取消订阅。

事件总线和pubsub消息订阅其实非常像，一般在vue中都是使用事件总线，不使用pubsub消息订阅。
